## Summary

V3.0 is the largest feature release in the platform's history. It ships six new capabilities, a unified team operating plan, a milestone backup workflow, and the foundation for Q1 deploy-stability hardening. The release was developed across two sessions and verified against a clean production build before deploy.

## Features

### 1. Email Editor 600px Preview
The admin email campaign editor now renders a 600-pixel-wide preview that matches the rendered email exactly. Previously, the preview used a fluid width that did not reflect how the email would appear in Gmail/Outlook. The new preview uses the same CSS as the sent email, ensuring WYSIWYG editing.

### 2. Click-to-Edit on Every List
Every member, registrant, and speaker name in the admin tables is now click-to-edit. Clicking a name opens the editor dialog for that record, eliminating the need to scroll to the rightmost "Edit" button. This is a significant UX improvement for tables with many columns.

### 3. Explicit Edit Button on Every Row
In addition to click-to-edit, every row in every admin table now has an explicit "Edit" button as the rightmost column. This makes the editing affordance discoverable for users who don't realize names are clickable.

### 4. Combobox Company Field
The member editor's company field is now a combobox that suggests existing companies as you type, or accepts a new company name. This reduces typos and makes it easier to filter by company later. The list of existing companies comes from the new `/api/admin/members/companies` endpoint.

### 5. Speaker Image Slideshow + Drag-to-Reorder
The four-picture cap on speaker images has been removed. Speakers can now have any number of images. The admin speaker editor shows a slideshow viewer with:
- Thumbnail with `1/N` counter (e.g., `1/7`) in the top-right corner
- Click thumbnail to open the slideshow dialog
- Drag-and-drop reordering (powered by dnd-kit)
- Reorder persists via the new `POST /api/images/reorder` endpoint

### 6. CSV/XLS Bulk Import
Admins can now bulk-import members and event registrants from any CSV, XLS, or XLSX file directly from the admin tables. Features include:
- "Import CSV/XLS" button in the toolbar of both Members and Registrants tables
- File picker accepts `.csv`, `.xls`, `.xlsx`
- Downloadable CSV template with correct headers and an example row
- Three-card summary after import: New / Updated / Skipped counts
- Per-row error report for the first 50 skipped rows
- Registrants import includes a target-event picker
- Existing rows are updated (upserted), not duplicated — re-importing is idempotent
- All imports are audit-logged with `importSource: 'bulk-import:filename'`

## API Endpoints Added

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/admin/members/companies` | GET | Returns list of existing company names for the combobox |
| `/api/admin/members/bulk-import` | POST | Multipart upload — parses CSV/XLS, upserts members |
| `/api/admin/members/import-template` | GET | Returns downloadable CSV template |
| `/api/admin/registrants/bulk-import` | POST | Multipart upload — parses CSV/XLS, upserts registrants for an event |
| `/api/admin/registrants/import-template` | GET | Returns downloadable CSV template |
| `/api/images/reorder` | POST | Reorders speaker images via drag-and-drop |

## Database Schema Changes

None. V3.0 added no new tables and no new columns. The `User.role` column comment was updated to reflect the upcoming four-role RBAC model (SUPER_ADMIN/ADMIN/CO_HOST/MEMBER), but the column type and default value are unchanged. No migration required.
