## About gofile.io

- **Free, anonymous, no account required** — files are stored on EU/NA servers
- **No expiration** for active files (files that haven't been downloaded in
  a long time *may* eventually be pruned, but there's no hard expiry)
- **Guest token** is required to manage (rename, delete) the files later.
  Keep this file safe — losing the guest token means losing management
  access (the file will still be downloadable, but you can't delete it)
- **No rate limits** on downloads

## To delete a file later

```bash
curl -X DELETE "https://api.gofile.io/contents/<fileId>?wt=<guestToken>"
```

Example for the V3.1 tarball:
```bash
curl -X DELETE "https://api.gofile.io/contents/82b2f59b-5f25-42bd-a388-81b2f2de131a?wt=vw6n4bpytqE8ZXXrf3f0QKFyHanAEkev"
```

4. `bun install`
5. Set `DATABASE_URL` env var to point to the production Postgres
   (or a snapshot — Vercel Postgres → Storage → Snapshots)
6. `bun run build`
7. `vercel link --yes` (re-link to project `prj_aoKtARAel8wlmcIlLRjjSPKshMLA`)
8. `vercel deploy --prod --yes`
